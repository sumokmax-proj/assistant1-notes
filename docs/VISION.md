# VISION.md — 프로젝트 목표 및 원칙

---

## 한 줄 목표

> **AI가 스스로 돈을 버는 방법을 안전하게 탐색하고, 실행하고, 지속적으로 개선한다.**

---

## 왜 이 프로젝트인가

- 현재 AI는 뛰어난 사고 능력을 갖고 있지만, 실제 세계에서 자립할 수단이 없다
- 특정 인프라(Conway 등)에 종속되지 않고 범용적으로 자립 가능한 구조를 탐구한다
- 인간(Operator)과 AI(Claude)가 협력하는 새로운 일하는 방식을 직접 실험한다

---

## 역할 정의

| 역할 | 담당 | 책임 범위 |
|---|---|---|
| **Operator (당신)** | 인간 파트너 | 법적 주체, 계좌/지갑, 최종 실행 승인, 자본 투입 |
| **Navigator (Claude)** | AI 파트너 | 전략 수립, 코드 작성, 분석, 문서 관리, 점검 |

---

## 단계별 성공 기준

### Phase 0 — 기반 구축 (현재)
- [x] 협업 구조 합의
- [x] 문서 체계 수립
- [ ] 첫 번째 수익 모델 선정
- [ ] 실행 환경 준비

### Phase 1 — 첫 번째 수익 검증
- 목표: **최소 $1** 외부 수익 발생
- 조건: 초기 투자 $100 미만
- 기간: 최대 8주
- 성공 기준: 한 명 이상의 외부인이 자발적으로 결제

### Phase 2 — 시스템화
- 목표: **월 $100** 반복 수익
- 조건: 인간 개입을 최소화한 자동화 구조
- 기간: Phase 1 이후 2~3개월

### Phase 3 — 자립
- 목표: **수익 > 운영비용** (흑자 전환)
- 조건: AI 추론 비용 + 인프라 비용을 수익으로 충당
- 기간: Phase 2 이후 지속

## 핵심 원칙

### 1. 안전 우선
- 해를 끼치는 방법으로는 절대 수익을 추구하지 않는다
- 스팸, 사기, 무단 크롤링, 저작권 침해는 금지
- 불확실한 상황에서는 행동하기 전에 Operator에게 확인

### 2. 투명성
- 모든 시도와 실패를 RESULTS_LOG에 솔직하게 기록
- AI는 자신이 AI임을 숨기지 않는다
- 비용과 수익을 정확히 추적한다

### 3. 작게 시작, 빠르게 배우기
- 완벽한 계획보다 빠른 실험을 우선한다
- 실패는 데이터다. 실패에서 구체적인 교훈을 추출한다
- 한 번에 하나의 수익 모델만 집중한다

### 4. 인간 승인 게이트
- 돈과 보안에 관한 모든 행동은 Operator 승인 후 실행
- Claude는 제안하고, Operator가 결정한다

### 5. Conway 비종속
- 특정 플랫폼/인프라에 종속되지 않는 설계를 지향한다
- 모든 핵심 로직은 이식 가능하게 설계한다

## 금지 사항

- 스팸성 콘텐츠 대량 생산
- 다른 사람의 콘텐츠 무단 복제/판매
- 사용자를 속이는 서비스
- 투기적 자산(밈코인 등)에 예산 투입
- Operator 승인 없이 결제 또는 계정 생성
